**Promise**

What is a Promise?
A Promise is an object that represents the eventual completion (or failure) of an asynchronous operation and the resulting value of that operation. It is like a promise in real life: it can be fulfilled or rejected in the future.

States of a Promise
Pending:

The operation is still in progress and the result is not yet available.
Fulfilled:

The operation completed successfully, and we have the result.
Rejected:

The operation failed, and we have a reason for the error.
A Promise can only be in one state at a time, and once it is fulfilled or rejected, it does not change.

Object which represents the completion of an asynchronous operation and its return value

**SaaS**

SaaS stands for Software as a Service, which means "Software as a Service". It is a software distribution model where applications are hosted by a provider (usually in the cloud) and accessed by users via the internet. In this model, you do not need to install the software on your local computer or server, as everything works remotely.

Main characteristics of SaaS:
Cloud-based: The software is hosted online and can be accessed from anywhere with an internet connection.
Subscription model: Typically, you pay a monthly or annual fee to use the service.
Maintenance and updates managed by the provider: The supplier takes care of upgrades, security and maintenance, leaving the user free from these concerns.
Multiplatform access: Can be used on different devices, such as computers, tablets or smartphones.
Common examples of SaaS:
Google Workspace (Gmail, Google Drive, Google Docs)
Microsoft 365 (Word, Excel, Outlook online)
Slack (business communication)
Shopify (e-commerce platform)
Zoom (video conferencing)
This is a widely used model today because it facilitates the adoption of modern technologies, without the need for robust infrastructure on the part of the client.
